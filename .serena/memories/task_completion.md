# 작업 완료 시 수행 절차

## 필수 체크리스트

### 1. 코드 품질
- [ ] 컴파일 에러 없음 확인
- [ ] 네이밍 컨벤션 준수 (PascalCase, _camelCase 등)
- [ ] 네임스페이스 규칙 준수 (`Sc.{폴더명}`)
- [ ] 파일당 클래스 1개 원칙

### 2. Assembly 규칙
- [ ] 새 Assembly 생성 시 `IsExternalInit.cs` 추가
- [ ] Assembly 의존성 순환 참조 없음 확인
- [ ] asmdef 참조 올바르게 설정

### 3. 테스트
- [ ] 관련 단위 테스트 통과
- [ ] Unity Test Runner에서 테스트 실행

### 4. PROGRESS.md 업데이트 (필수)
- [ ] 작업 시작 시 체크리스트에 항목 추가
- [ ] 단계 완료 시 `- [ ]` → `- [x]` 체크
- [ ] 시스템 상태 테이블 업데이트 (⬜ → 🔨 → ✅)

### 5. 문서 동기화
- [ ] 스펙 문서와 구현 일치 확인
- [ ] 새 시스템의 경우 스펙 문서 작성/업데이트

### 6. 커밋
- [ ] 제목: 영어 (동사 원형 시작)
- [ ] 본문: 한국어
- [ ] AI 관련 문구 추가 금지 (Co-Authored-By 등)

## 작업 흐름 요약
```
1. PROGRESS.md 확인
2. 스펙 문서 확인/작성
3. 구현
4. 테스트
5. PROGRESS.md 업데이트
6. 커밋
```
